
**User Story 11.6**

1.  **Titre :** Intégrer les Diagrammes de Points (Stitch Charts) dans l'Affichage Web et l'Export PDF
2.  **Objectif :** En tant qu'utilisateur consultant un patron qui utilise un diagramme de points, je veux voir ce diagramme clairement affiché dans l'application web (au sein du `PatternViewer`) et inclus de manière lisible dans le document PDF exporté.
3.  **Description :** Cette User Story prend la structure de données du diagramme générée par US 11.5 et s'occupe de son rendu visuel effectif tant sur la page web que dans le PDF. Cela répond directement à votre demande de s'assurer de l'affichage au niveau de l'application web.
4.  **Exigences Fonctionnelles :**
    * FR1 : **Affichage Web :**
        * Dans le `PatternViewer` (US 9.1), lorsqu'un composant de patron utilise un diagramme de points, celui-ci doit être rendu graphiquement.
        * Chaque case de la grille du diagramme doit afficher le symbole graphique correspondant (issu de `standard_stitch_symbols.graphic_asset_url`).
        * La numérotation des rangs et des mailles (pour une répétition) doit être affichée le long des axes du diagramme.
        * La légende spécifique au diagramme (symboles utilisés et leurs définitions) doit être affichée à proximité du diagramme.
        * Les cases "pas de maille" doivent être clairement indiquées (par exemple, grisées ou vides).
        * Le sens de lecture des rangs doit être indiqué (par ex. flèches ou numérotation des rangs sur les bons côtés pour le tricot à plat).
    * FR2 : **Export PDF :**
        * Les diagrammes de points rendus graphiquement (avec leur légende et numérotation) doivent être inclus dans le document PDF généré (US 9.2) pour les sections de patron concernées.
        * Le diagramme dans le PDF doit être de haute qualité pour une bonne lisibilité à l'impression.
    * FR3 : Le rendu des diagrammes doit être configurable pour s'adapter à différents thèmes ou préférences d'affichage (ex: taille des cases, couleurs pour "pas de maille").
5.  **Guide d'Implémentation Technique :**
    * **Rendu Web :**
        * **Approche SVG (recommandée) :** Utiliser JavaScript pour générer dynamiquement un élément SVG représentant le diagramme. Chaque case de la grille devient un `<rect>` ou un `<g>` groupe, et chaque symbole est rendu en utilisant un `<image>` (pointant vers le fichier SVG/PNG du symbole) ou en dessinant le symbole directement avec des primitives SVG si les symboles sont simples.
        * **Approche Canvas :** Similaire, mais dessiner sur un élément `<canvas>`. Peut être moins flexible pour l'interactivité future ou la sélection de texte.
        * **Approche Table HTML :** Utiliser une table HTML où chaque cellule contient une image du symbole. Moins élégant mais potentiellement plus simple pour un rendu de base.
        * Un composant Vue/React/Angular dédié (`StitchChartDisplay.vue`) serait responsable de ce rendu.
    * **Rendu PDF :**
        * Si le rendu web utilise SVG, le SVG peut souvent être converti en image (PNG/JPEG) ou directement incorporé dans le PDF si la bibliothèque PDF le supporte (beaucoup le font, comme ReportLab pour Python, ou via des intermédiaires pour iText/PDFBox).
        * La bibliothèque PDF choisie en US 9.2 doit être capable d'intégrer des images ou des graphiques vectoriels.
    * **Logique/Traitement :**
        * Le `PatternViewer` reçoit la structure de données du diagramme (de US 11.5, incluse dans les données du patron assemblé par US 9.1).
        * Il passe ces données au composant `StitchChartDisplay.vue` pour le rendu web.
        * Pour le PDF, le `PdfExportService` (US 9.2) recevrait également ces données et utiliserait une logique similaire ou une image du diagramme pour l'incorporer.
    * **Points d'Intégration :**
        * Consomme la sortie structurée de US 11.5 (`StitchChartGeneratorService`).
        * S'intègre dans le `PatternViewer.vue` (US 9.1) pour l'affichage web.
        * S'intègre dans le `PdfExportService.java` (US 9.2) pour l'inclusion PDF.
6.  **Critères d'Acceptation (Validation Fonctionnelle) :**
    * AC1 : Un diagramme de point pour un motif de torsade simple s'affiche correctement dans le `PatternViewer` de l'application web, avec les symboles, la numérotation des rangs/mailles et la légende.
    * AC2 : Le même diagramme est inclus de manière lisible et correcte dans le PDF exporté du patron.
    * AC3 : Les symboles "pas de maille" sont clairement différenciés des cases contenant des symboles d'action.
    * AC4 : Le sens de lecture des rangs est intuitivement indiqué sur le diagramme (web et PDF).
7.  **Stratégie de Test :**
    * **Tests Unitaires :**
        * Pour le composant de rendu web (`StitchChartDisplay.vue`):
            * Tester le rendu d'une grille simple (ex: 2x2) avec des symboles connus.
            * Tester l'affichage correct de la numérotation des rangs et des mailles.
            * Tester l'affichage de la légende.
            * Tester le rendu des cases "pas de maille".
        * Pour la partie PDF:
            * Tester la fonction d'intégration d'un diagramme (en tant qu'image ou SVG) dans un document PDF.
    * **Tests d'Intégration :**
        * Vérifier que la structure de données du diagramme générée par `StitchChartGeneratorService` est correctement interprétée et rendue par `StitchChartDisplay.vue`.
        * Générer un patron complet contenant un diagramme et vérifier son affichage dans le `PatternViewer` et son inclusion dans le PDF.
    * **Tests de Non-Régression :**
        * S'assurer que les patrons qui n'utilisent pas de diagrammes s'affichent toujours correctement (web et PDF).
        * Vérifier que les autres aspects de l'affichage du patron (texte, schémas statiques de US 9.3) ne sont pas affectés.
8.  **Hypothèses/Pré-conditions :**
    * US 11.5 est implémenté et fournit une structure de données claire et correcte pour les diagrammes.
    * Les fichiers graphiques (SVG/PNG) pour chaque symbole standard sont disponibles et accessibles.
    * La bibliothèque PDF utilisée supporte l'intégration d'images ou de SVG.
9.  **Composants Système Impactés (Illustratif) :**
    * UI : Nouveau composant `StitchChartDisplay.vue`. Modification de `PatternViewer.vue` (US 9.1).
    * Backend : Modification du `PdfExportService.java` (US 9.2) pour gérer l'intégration des diagrammes.
    * Potentiellement, un service côté serveur pour convertir SVG en image si nécessaire pour le PDF, ou si le rendu SVG est fait côté serveur.
