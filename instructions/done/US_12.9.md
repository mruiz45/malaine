**User Story 12.9**

1.  **Titre :** Implémenter un Outil Interactif de Prévisualisation/Planification de l'Assemblage des Pièces du Vêtement en 2D
2.  **Objectif :** En tant qu'utilisateur, après que les schémas 2D de toutes les pièces de mon vêtement ont été générés (US 9.3), je veux pouvoir visualiser ces pièces sur un espace de travail interactif où je peux voir comment elles sont censées s'assembler (par exemple, quelles bordures se cousent ensemble), afin de mieux comprendre la construction globale avant de commencer à tricoter/crocheter ou à assembler.
3.  **Description :** Cette User Story vise à créer une interface où les schémas 2D des différentes pièces du patron (générés en US 9.3 et potentiellement améliorés avec des détails de façonnage) peuvent être affichés ensemble. L'utilisateur pourrait voir des indications sur les coutures ou les points d'assemblage. Une version de base pourrait afficher les pièces dans une disposition typique "à plat" avec des annotations sur les bords à joindre. Une version plus avancée pourrait permettre un léger "glisser-déposer" pour simuler l'assemblage. Cela s'inscrit dans la section 5.4 du PDF ("Interface Utilisateur pour la Visualisation").
4.  **Exigences Fonctionnelles :**
    * FR1 : Le système doit récupérer les schémas 2D (générés en US 9.3, par exemple en format SVG) et les dimensions clés pour tous les composants principaux du vêtement défini (ex: devant, dos, manches, col).
    * FR2 : Implémenter un nouvel espace de travail UI ("Vue d'Assemblage 2D").
    * FR3 : Dans cette vue, afficher les schémas 2D des pièces disposées de manière logique (par exemple, dos et devant côte à côte, manches à proximité des emmanchures).
    * FR4 : Mettre en évidence (par exemple, par des lignes de couleur, des numéros, ou des icônes) les bords des pièces qui doivent être assemblés/cousus ensemble. Par exemple, le haut d'une manche avec l'emmanchure du corps.
    * FR5 : En cliquant sur un bord ou une indication d'assemblage, des informations supplémentaires pourraient s'afficher (ex: "Coudre le haut de la manche à l'emmanchure du dos et du devant.").
    * FR6 : (Optionnel - version plus avancée) Permettre à l'utilisateur de "glisser" légèrement les pièces pour les rapprocher et mieux visualiser l'alignement des bords à assembler. Cela ne serait pas une simulation physique exacte mais une aide visuelle.
    * FR7 : Cette vue doit être accessible depuis le `PatternViewer` (US 9.1).
5.  **Guide d'Implémentation Technique :**
    * **Modèles de Données :**
        * Entrée : Les données du patron assemblé (de US 9.1), qui incluent les références aux schémas SVG (de US 9.3) et les dimensions calculées pour chaque pièce.
        * Pour identifier les bords à assembler, le `CorePatternCalculationEngine` ou le `PatternAssemblerService` (US 9.1) devrait enrichir les données de chaque composant avec des métadonnées sur ses bords (ex: `{"edge_id": "sleeve_cap_edge", "connects_to_component_key": "body_panel", "connects_to_edge_id": "armhole_edge", "length_cm": 40}`).
    * **Logique/Traitement (Frontend) :**
        * Le composant UI de la "Vue d'Assemblage 2D" recevrait la liste des schémas des pièces et leurs métadonnées d'assemblage.
        * **Rendu des pièces :** Afficher les SVG des schémas. Chaque SVG pourrait être rendu dans son propre conteneur déplaçable (si le glisser-déposer est implémenté).
        * **Indication des assemblages :**
            * Dessiner des lignes ou des icônes entre les bords correspondants des SVG. Cela pourrait être fait en superposant un autre SVG ou en utilisant des éléments HTML positionnés dynamiquement.
            * La logique pour déterminer les coordonnées des points de connexion sur les SVG des pièces sera nécessaire.
        * **Interactivité (si glisser-déposer) :** Utiliser des bibliothèques JavaScript de glisser-déposer (ex: interact.js, Draggable) ou les API natives HTML5.
    * **Points d'Intégration :**
        * Utilise les schémas 2D de US 9.3 et les données d'assemblage du patron de US 9.1.
        * S'intègre comme une nouvelle vue ou un onglet dans le `PatternViewer.vue`.
6.  **Critères d'Acceptation (Validation Fonctionnelle) :**
    * AC1 : Pour un patron de pull simple (dos, devant, 2 manches), la Vue d'Assemblage 2D affiche les 4 schémas de pièces.
    * AC2 : Les bords correspondants (ex: coutures d'épaule du devant avec dos, côtés du devant avec côtés du dos, emmanchures avec têtes de manches) sont clairement indiqués comme devant être assemblés.
    * AC3 : En cliquant sur une indication d'assemblage, une infobulle ou un message simple explique la connexion (ex: "Assembler le côté A du Devant avec le côté A du Dos").
    * AC4 : (Si glisser-déposer implémenté) L'utilisateur peut déplacer légèrement les pièces pour mieux visualiser les connexions.
7.  **Stratégie de Test :**
    * **Tests Unitaires (Frontend) :**
        * Tester le composant qui rend un seul schéma de pièce SVG.
        * Tester la logique qui dessine les indicateurs d'assemblage entre deux points/bords.
        * Tester la logique de glisser-déposer si implémentée.
        * Tester l'affichage des infobulles d'information.
    * **Tests d'Intégration (Frontend) :**
        * Charger les données d'un patron complet et vérifier que toutes les pièces sont affichées et que toutes les connexions d'assemblage attendues sont correctement indiquées.
        * Tester l'interaction complète (clics, glisser-déposer si applicable) dans la vue.
    * **Tests de Non-Régression :**
        * S'assurer que l'affichage standard du patron dans le `PatternViewer` (texte, diagrammes, schémas individuels) n'est pas affecté.
8.  **Hypothèses/Pré-conditions :**
    * Les schémas 2D (US 9.3) sont disponibles (par exemple, en format SVG) et le système connaît leurs dimensions clés.
    * Le moteur de calcul ou l'assembleur de patron (US 9.1) peut fournir des métadonnées sur les bords de chaque pièce qui doivent être assemblés. Définir ces "bords nommés" ou "points de connexion" sur chaque pièce est une tâche importante.
    * Pour la version initiale, une simulation physique ou une détection de collision n'est pas attendue pour le glisser-déposer.
9.  **Composants Système Impactés (Illustratif) :**
    * UI (Frontend) : Nouveau composant majeur `GarmentAssemblyViewer.vue`, potentiellement des sous-composants pour les pièces et les connecteurs. Modification de `PatternViewer.vue` pour inclure cette nouvelle vue.
    * Backend : Le `PatternAssemblerService` ou le `CorePatternCalculationEngine` pourrait avoir besoin d'être étendu pour générer les métadonnées d'assemblage des bords (par exemple, en identifiant les longueurs de couture correspondantes).
