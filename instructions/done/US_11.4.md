
**User Story 11.4**

1.  **Titre :** Générer les Instructions Textuelles pour Emmanchures Complexes (Arrondies, Raglan)
2.  **Objectif :** En tant que générateur d'instructions, après que le moteur de calcul a déterminé le planning de façonnage pour une emmanchure complexe (US 11.3), je dois traduire ce planning en instructions textuelles claires, étape par étape, pour le panneau de corps et potentiellement pour la tête de manche.
3.  **Description :** Cette User Story s'appuie sur la sortie de US 11.3 (`armhole_shaping_schedule`). Elle convertit les actions de façonnage (rabattre, diminuer) pour les emmanchures en langage de patron standard.
4.  **Exigences Fonctionnelles :**
    * FR1 : Le système doit prendre en entrée le `armhole_shaping_schedule` pour le corps et/ou la manche.
    * FR2 : Générer des instructions pour les mailles rabattues à la base de l'emmanchure (ex: "Au début des 2 prochains rangs, rabattre 3 mailles.").
    * FR3 : Pour chaque étape de diminution dans le planning de l'emmanchure (corps ou raglan) :
        * Préciser sur quel type de rang (endroit/envers) l'action doit être effectuée.
        * Utiliser une terminologie standard pour les diminutions.
        * Indiquer clairement la répétition.
    * FR4 : Pour les raglans, les instructions doivent souvent spécifier de travailler jusqu'à X mailles avant un marqueur de raglan, effectuer une diminution, glisser le marqueur, effectuer une autre diminution (si c'est une double diminution de raglan), puis continuer. Les instructions pour les manches et le corps doivent être synchronisées si elles sont tricotées en même temps (par exemple, pour un raglan tricoté d'une pièce). *Pour des pièces tricotées séparément, les instructions sont par pièce.*
    * FR5 : (Si le façonnage de la tête de manche est inclus) Générer des instructions pour les augmentations formant la base de la tête de manche, puis les diminutions pour le sommet.
    * FR6 : Indiquer le nombre de mailles restantes après les étapes clés.
5.  **Guide d'Implémentation Technique :**
    * **Modèles de Données :**
        * Entrée : La structure `armhole_shaping_schedule` (et `sleeve_cap_shaping_schedule` si applicable) de US 11.3.
        * Sortie : Instructions textuelles à intégrer dans les instructions globales du composant.
    * **Logique/Traitement :**
        * Similaire à US 11.2 pour les encolures : parcourir le planning de façonnage et générer des phrases d'instruction.
        * Pour les raglans, la logique de description des diminutions par rapport aux marqueurs est spécifique.
        * La gestion des instructions pour les têtes de manche montées est particulièrement technique, impliquant souvent des augmentations rapides suivies de diminutions progressives puis rapides.
    * **Points d'Intégration :**
        * Consomme la sortie de US 11.3.
        * S'intègre dans le `InstructionGeneratorService.java`.
6.  **Critères d'Acceptation (Validation Fonctionnelle) :**
    * AC1 : Pour un planning d'emmanchure arrondie (rabattre 3m à la base, puis diminuer 2m, puis 1m x N fois), les instructions textuelles reflètent précisément cette séquence.
    * AC2 : Pour un planning de raglan (diminuer 1m de chaque côté de la ligne de raglan tous les 2 rangs), les instructions sont claires et précises.
    * AC3 : La terminologie est correcte pour le tricot/crochet.
7.  **Stratégie de Test :**
    * **Tests Unitaires :**
        * Tester la génération de phrases pour les diminutions d'emmanchure arrondie.
        * Tester la génération de phrases pour les diminutions de raglan (par rapport à un marqueur).
        * Tester la génération de phrases pour les augmentations de tête de manche.
    * **Tests d'Intégration :**
        * Envoyer un `armhole_shaping_schedule` complet au service et vérifier la cohérence des instructions générées.
    * **Tests de Non-Régression :**
        * S'assurer que la génération d'instructions pour les encolures (US 11.2) et autres formes n'est pas affectée.
8.  **Hypothèses/Pré-conditions :**
    * US 11.3 est implémenté et fournit des plannings de façonnage précis.
    * Le `craft_type` est connu.
9.  **Composants Système Impactés (Illustratif) :**
    * Backend : `InstructionGeneratorService.java` (extension significative).
