
**User Story 12.6**

1.  **Titre :** Générer les Instructions Textuelles pour Châles Triangulaires Basiques
2.  **Objectif :** En tant que générateur d'instructions, après que le moteur de calcul a déterminé le montage et le planning d'augmentations/diminutions pour un châle triangulaire (US 12.5), je dois traduire ces informations en instructions textuelles claires, étape par étape, pour que l'utilisateur puisse tricoter/crocheter le châle.
3.  **Description :** Cette User Story s'appuie sur la sortie de US 12.5 (`triangular_shawl_calculations`). Elle génère les instructions spécifiques à la méthode de construction choisie pour le châle triangulaire.
4.  **Exigences Fonctionnelles :**
    * FR1 : Prendre en entrée la structure `triangular_shawl_calculations` (de US 12.5).
    * FR2 : Pour la construction **Top-Down, départ central** :
        * Instructions pour le montage initial (ex: garter tab cast-on si utilisé, ou montage simple).
        * Instructions pour le rang de mise en place (setup row/round) avec placement des marqueurs pour la maille centrale et les bords.
        * Instructions pour les rangs/tours d'augmentation (ex: "Rang Endroit (Augmentation) : K1, M1R, tricoter jusqu'à 1m avant le marqueur central, M1R, K1 (maille centrale), M1L, tricoter jusqu'au dernier marqueur/maille, M1L, K1.").
        * Instructions pour les rangs/tours envers (souvent tricotés unis).
        * Indiquer de répéter ces X rangs (ex: 2 rangs) jusqu'à la fin.
    * FR3 : Pour la construction **Pointe à pointe** :
        * Instructions pour le montage initial.
        * Instructions pour les rangs d'augmentation sur un seul bord (ex: "Rang Endroit : KFB ou M1 dans la première maille, tricoter jusqu'à la fin.").
        * Instructions pour les rangs envers.
        * Indiquer quand passer à la phase de diminution.
        * Instructions pour les rangs de diminution sur ce même bord.
    * FR4 : Pour la construction **Du bas vers le haut** :
        * Instructions pour le montage du grand nombre de mailles.
        * Instructions pour les rangs de diminution sur les deux bords (ex: "Rang Endroit : K1, SSK, tricoter jusqu'aux 3 dernières mailles, K2tog, K1.").
        * Instructions pour les rangs envers.
    * FR5 : Indiquer le nombre de mailles après des étapes clés ou à intervalles réguliers si utile.
    * FR6 : Inclure des instructions pour rabattre les mailles à la fin.
5.  **Guide d'Implémentation Technique :**
    * **Modèles de Données :**
        * Entrée : La structure `triangular_shawl_calculations` de US 12.5.
        * Sortie : Instructions textuelles à intégrer dans la section "Châle" du patron.
    * **Logique/Traitement :**
        * Utiliser des modèles de phrases pour chaque action et méthode de construction.
        * La logique pour les augmentations (M1L, M1R, KFB, jeté) ou diminutions (K2tog, SSK) doit être standardisée.
        * Gérer la terminologie tricot/crochet. Pour le crochet, les augmentations/diminutions se font souvent en ajoutant/sautant des mailles ou en travaillant plusieurs mailles dans une seule.
    * **Points d'Intégration :**
        * Consomme la sortie de US 12.5.
        * S'intègre dans le `InstructionGeneratorService.java`.
6.  **Critères d'Acceptation (Validation Fonctionnelle) :**
    * AC1 : Les instructions pour un châle top-down (départ central) sont claires concernant le montage, le placement des marqueurs, les 4 augmentations par rang endroit, et les rangs envers.
    * AC2 : Les instructions pour un châle pointe à pointe détaillent correctement la phase d'augmentation puis la phase de diminution.
    * AC3 : Les instructions pour un châle du bas vers le haut sont correctes pour le montage initial et les diminutions régulières.
    * AC4 : La terminologie est appropriée et les instructions sont faciles à suivre pour un utilisateur connaissant les techniques de base.
7.  **Stratégie de Test :**
    * **Tests Unitaires :**
        * Tester la génération de la phrase pour un rang d'augmentation typique d'un châle top-down.
        * Tester la génération de la phrase pour un rang de diminution typique d'un châle bottom-up.
        * Valider la description des augmentations/diminutions spécifiques (M1L, KFB, SSK, etc.).
    * **Tests d'Intégration :**
        * Envoyer une structure `triangular_shawl_calculations` complète au service et vérifier la séquence et la justesse des instructions générées pour chaque méthode de construction.
    * **Tests de Non-Régression :**
        * S'assurer que la génération d'instructions pour les pulls et autres types de vêtements n'est pas affectée.
8.  **Hypothèses/Pré-conditions :**
    * US 12.5 est implémenté et fournit des calculs précis.
    * Le `craft_type` est connu.
    * Les types d'augmentations/diminutions utilisés sont standards (une amélioration future pourrait permettre à l'utilisateur de choisir ses méthodes préférées).
9.  **Composants Système Impactés (Illustratif) :**
    * Backend : `InstructionGeneratorService.java` (extension pour la logique des châles triangulaires).
