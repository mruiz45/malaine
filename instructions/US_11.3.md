
**User Story 11.3**

1.  **Titre :** Implémenter les Calculs de Mise en Forme pour Emmanchures Complexes (Arrondies, Raglan)
2.  **Objectif :** En tant que moteur de calcul de patron, étant donné les paramètres d'une emmanchure (type : arrondie/montée, raglan ; profondeur et largeur désirées) et l'échantillon, je dois calculer la séquence précise de diminutions (et potentiellement de mailles rabattues) pour former correctement l'emmanchure sur un panneau de corps et sur la tête de manche correspondante.
3.  **Description :** Similaire à US 11.1 pour les encolures, cette User Story se concentre sur le façonnage des emmanchures, qui est crucial pour l'ajustement des manches. Cela inclut les emmanchures classiques arrondies pour manches montées et les diminutions diagonales caractéristiques des raglans. Cela correspond à une partie avancée de la section 4.2 du PDF ("Gestion des Augmentations et Diminutions").
4.  **Exigences Fonctionnelles :**
    * FR1 : Le système doit accepter en entrée : le type d'emmanchure ('arrondie_montée', 'raglan'), la profondeur totale de l'emmanchure, la largeur que l'emmanchure doit "retirer" du panneau de corps, et l'échantillon. Pour les raglans, la longueur de la ligne de raglan est aussi un paramètre clé.
    * FR2 : Pour une **emmanchure arrondie (manche montée)** :
        * Calculer le nombre de mailles à rabattre à la base de l'emmanchure (de chaque côté du panneau de corps).
        * Calculer le nombre de mailles restantes à diminuer de chaque côté pour former la courbe de l'emmanchure.
        * Déterminer une série de diminutions progressives (similaire à l'encolure arrondie) pour créer la courbe.
        * Calculer le façonnage correspondant pour la tête de manche (augmentations pour former la courbe de la tête de manche, puis diminutions pour la partie supérieure). *Note : La tête de manche est complexe et pourrait être une US séparée ou une partie avancée de celle-ci.* Pour cette US, nous pouvons nous concentrer sur le façonnage de l'emmanchure sur le corps.
    * FR3 : Pour une **emmanchure raglan** :
        * Calculer le nombre de mailles à rabattre à la base de l'emmanchure (si applicable, souvent quelques mailles pour l'aisance sous le bras).
        * Calculer le nombre total de diminutions nécessaires le long de la ligne de raglan pour atteindre la largeur d'encolure/épaule souhaitée.
        * Déterminer un taux de diminution régulier (ex: diminuer 1 maille de chaque côté de la ligne de raglan tous les X rangs, Y fois). Cela doit se faire simultanément sur le corps (devant et dos) et sur les manches.
    * FR4 : La sortie doit être une structure de données décrivant la séquence des actions (rabattre, diminuer) par rang ou groupe de rangs pour chaque emmanchure (corps) et, si inclus, pour la tête de manche.
5.  **Guide d'Implémentation Technique :**
    * **Modèles de Données :**
        * Entrée : Fait partie de la structure d'entrée du moteur de calcul (US 6.1), spécifiquement pour les composants "panneau de corps" et "manche", où les attributs d'emmanchure (issus de US 4.3 et 4.5, par exemple le choix de "manche montée" ou "raglan") sont définis.
        * Sortie (dans la section "calculations" pour les composants concernés) :
            ```json
            "armhole_shaping_schedule": { // Pour le panneau de corps
              "type": "rounded_set_in", // "raglan"
              "base_bind_off_stitches": 3, // Mailles rabattues à la base
              "shaping_details": [
                {"action": "decrease", "stitches": 2, "on_row_from_start_of_shaping": 3, "side_of_fabric": "RS"},
                {"action": "decrease", "stitches": 1, "on_row_from_start_of_shaping": 5, "side_of_fabric": "RS", "repeats": 4, "every_x_rows": 2}
              ],
              "total_rows_for_shaping": 24, // Hauteur de l'emmanchure
              "final_stitches_at_shoulder_edge": 5 // Mailles restantes avant l'épaule
            },
            "sleeve_cap_shaping_schedule": { // Pour la manche (si incluse dans cette US)
              "type": "rounded_set_in", // "raglan_sleeve_top"
              // ... structure similaire avec augmentations puis diminutions ...
            }
            ```
    * **Logique/Traitement :**
        * Convertir les dimensions en mailles/rangs.
        * Emmanchure arrondie : Utiliser des ratios standards (ex: 1/3 mailles rabattues à la base, puis diminutions progressives) ou une approche par courbe.
        * Raglan : Calculer la pente de la ligne de raglan. La fréquence des diminutions est souvent "diminuer 1m tous les 2 rangs" ou "tous les 4 rangs" de chaque côté des "marqueurs" de raglan (qui peuvent être des mailles spécifiques ou juste un point de référence).
        * La logique doit savoir à quel rang du panneau/manche le façonnage de l'emmanchure commence.
    * **Points d'Intégration :**
        * Consomme les paramètres de type de manche/construction (US 4.3, 4.5).
        * La sortie sera utilisée par le générateur d'instructions (US 11.4).
        * S'intègre au `CorePatternCalculationEngine.java`.
6.  **Critères d'Acceptation (Validation Fonctionnelle) :**
    * AC1 : Pour une emmanchure arrondie demandée de 22cm de profondeur, avec 3 mailles rabattues à la base, le système calcule une séquence de diminutions progressives pour le reste de la courbe sur le panneau de corps.
    * AC2 : Pour une ligne de raglan de 25cm de long qui doit réduire la largeur des mailles du corps de 15 mailles de chaque côté, le système calcule une fréquence de diminution appropriée (ex: diminuer 1m tous les X rangs).
    * AC3 : La structure de données en sortie (`armhole_shaping_schedule`) est correctement formatée.
7.  **Stratégie de Test :**
    * **Tests Unitaires :**
        * Tester la fonction de calcul des étapes de diminution pour une courbe d'emmanchure.
        * Tester la fonction de calcul du taux de diminution pour une ligne de raglan.
        * Vérifier la gestion des cas où la profondeur d'emmanchure est faible/importante.
    * **Tests d'Intégration :**
        * Vérifier que les paramètres de construction de manche/corps sont correctement utilisés pour déclencher le bon type de calcul d'emmanchure.
        * S'assurer que le `armhole_shaping_schedule` est compatible avec le générateur d'instructions.
    * **Tests de Non-Régression :**
        * S'assurer que les calculs d'encolure (US 11.1) et autres mises en forme ne sont pas affectés.
8.  **Hypothèses/Pré-conditions :**
    * L'échantillon est connu.
    * Les dimensions/types d'emmanchures sont fournis par l'utilisateur (via les outils de définition).
    * Le façonnage de la tête de manche pour les manches montées est complexe et pourrait nécessiter sa propre US ou une phase d'affinage de cette US si la première implémentation est simplifiée. Pour l'instant, on peut se concentrer sur l'emmanchure du corps.
9.  **Composants Système Impactés (Illustratif) :**
    * Backend : `CorePatternCalculationEngine.java` (ajout de modules/méthodes comme `ArmholeShapingCalculator.java`, `RaglanCalculator.java`).
